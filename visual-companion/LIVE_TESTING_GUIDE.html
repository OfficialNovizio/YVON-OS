Supabase Tables To Create

Run both SQL blocks in Supabase → SQL Editor

1. token_usage

CREATE TABLE IF NOT EXISTS token_usage (
  id                    uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  agent_id              text,
  route                 text NOT NULL,
  model                 text NOT NULL,
  input_tokens          integer DEFAULT 0,
  output_tokens         integer DEFAULT 0,
  cache_read_tokens     integer DEFAULT 0,
  cache_creation_tokens integer DEFAULT 0,
  cost_usd              numeric(10,6) DEFAULT 0,
  venture_id            text,
  created_at            timestamptz DEFAULT now()
);
CREATE INDEX IF NOT EXISTS idx_token_usage_created ON token_usage (created_at DESC);
CREATE INDEX IF NOT EXISTS idx_token_usage_agent   ON token_usage (agent_id, created_at DESC);

2. pending_changes (Code Hub)

CREATE TABLE IF NOT EXISTS pending_changes (
  id                uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  project_id        text NOT NULL,
  agent_id          text NOT NULL,
  agent_name        text,
  branch_name       text NOT NULL,
  commit_message    text NOT NULL,
  files             jsonb NOT NULL DEFAULT '[]',
  diff              text,
  status            text NOT NULL DEFAULT 'pending',
  requested_at      timestamptz DEFAULT now(),
  reviewed_at       timestamptz,
  review_note       text
);
CREATE INDEX IF NOT EXISTS idx_pending_changes_project
  ON pending_changes (project_id, status, requested_at DESC);

Deploy to Vercel + Connect Your Domain

ℹ️ You need a GitHub repo first

Vercel deploys from a GitHub (or GitLab/Bitbucket) repo. If YVON isn't pushed yet, do that first: git init → git remote add origin → git push

1
Push YVON to GitHub
Create a private repo at github.com/new, then: 
cd /path/to/YVON
git init
git add .
git commit -m "Initial YVON OS"
git remote add origin https://github.com/YOUR_USERNAME/yvon.git
git push -u origin main
2
Import to Vercel
Go to vercel.com/new → Import Git Repository → pick your YVON repo. Framework should auto-detect as Next.js. Leave root directory blank.
3
Add ALL environment variables
Before clicking Deploy, expand "Environment Variables" and add every key from your .env.local. Click "Add" for each. Do NOT skip any — missing vars cause silent failures.

ANTHROPIC_API_KEY NEXT_PUBLIC_SUPABASE_URL NEXT_PUBLIC_SUPABASE_ANON_KEY SUPABASE_SERVICE_ROLE_KEY RESEND_API_KEY BRIEFING_EMAIL APIFY_TOKEN YOUTUBE_API_KEY GOOGLE_SA_JSON CRON_SECRET NOVIZIO_IG_HANDLE
4
Deploy
Click Deploy. Build takes ~90 seconds. You'll get a free URL like yvon-stark.vercel.app. Test it first before connecting your domain.
5
Connect your custom domain
In Vercel: Project → Settings → Domains → "Add Domain" → type your domain (e.g. yvon.yourdomain.com or dashboard.yourdomain.com).

Vercel will show you DNS records to add. Two options:

Option A — Subdomain (recommended, e.g. dashboard.yourdomain.com):
Add a CNAME record at your DNS provider:
Type: CNAME
Name: dashboard
Value: cname.vercel-dns.com
Option B — Root domain (e.g. yourdomain.com):
Add two A records: 
Type: A  Name: @  Value: 76.76.21.21
Type: A  Name: @  Value: 76.76.21.9
DNS propagates in 10–60 minutes. SSL is automatic.
6
Lock down access (optional but recommended)
YVON has no auth layer — it's your private BI tool. Options to protect it:

Easiest: Vercel Password Protection (Vercel Pro plan) — one shared password, zero code changes
Free: Add Next.js middleware with a basic secret cookie check
Best: Add NextAuth.js with Google login — only your Gmail can access
