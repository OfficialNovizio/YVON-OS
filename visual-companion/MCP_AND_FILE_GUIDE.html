Part 1 — What Are FILES.md, COMMANDS.md, SKILLS.md?

How the Agent File System Works
Each agent in /departments/ has a folder with up to 5 support files. Together they form the agent's complete operating context — read by Claude Code at the start of a session. Think of each file as one layer of the agent's "brain at load time."
FilePurposeWho Reads ItWhat It Contains
MEMORY.md Long-term context — the single most important file. Pre-seeded with decisions; grows via append over sessions. Claude Code on every task start for that agent Completed task log, architecture decisions, rejected patterns, standing rules, domain-specific notes
FILES.md Data map — tells Claude Code exactly which files and Supabase tables this agent needs to read at session start, and which ones they must never touch. Claude Code when loading an agent session Must-read file list with paths and reasons. Supabase tables the agent queries. Files the agent must never modify.
COMMANDS.md Terminal operations — exact bash commands the agent runs, with error interpretation tables and handoff protocols. Claude Code when the agent needs to run verification steps Exact bash commands (e.g. npx tsc --noEmit, npm run lint). Error-to-fix mapping. QA report format. Handoff trigger conditions.
SKILLS.md Identity + responsibilities — the agent's full role definition, what they own, what they support, who they connect with, and routing keywords. Claude Code when clarifying scope or routing a task Agent identity table. Core responsibilities. Team connections. War Room routing keywords. What the agent explicitly does NOT do.
*-PRINCIPLES.md Deep rules — extended operating principles for agents with complex domains (Dev, Quinn, Priya, Mia). Optional deep reference. Claude Code when the task needs domain-specific depth For Dev: engineering standards, git workflow. For Quinn: full QA checklist. For Priya: feature spec templates. For Mia: component architecture rules.

Not every agent needs COMMANDS.md. Only agents who run terminal commands or have structured output protocols. Here's the rule:

FILES.md is the session-start checklist. It tells Claude Code what to read before touching anything. After the refactor, the formula is:

Part 2 — MCPs You Need by Agent

What Is an MCP?
MCP (Model Context Protocol) connectors give Claude Code direct, authenticated access to external services — without needing to call their APIs manually through your Next.js routes. Instead of "Raj writes a /api/youtube route to fetch data", an MCP means "Claude Code calls YouTube directly in the session". They unlock live data access during development sessions.

How to Install MCPs

Install via Claude Code Settings
MCPs are configured in Claude Code's settings file. For Cowork mode, they can be connected through the MCP registry. The four P1 MCPs (Supabase, GitHub, Vercel, GA4) will give you the biggest return on the smallest install effort — start there.
MCPKey ConfigWhere to Get Credentials
SupabaseSUPABASE_URL, SUPABASE_SERVICE_ROLE_KEYSupabase Dashboard → Project Settings → API
GitHubGITHUB_TOKEN (Personal Access Token with repo scope)GitHub → Settings → Developer settings → Personal access tokens
VercelVERCEL_TOKEN, TEAM_IDVercel Dashboard → Settings → Tokens
Google AnalyticsGOOGLE_SA_JSON (same service account you already have)Same credentials as your existing GOOGLE_SA_JSON env var
StripeSTRIPE_SECRET_KEYStripe Dashboard → Developers → API keys (use Restricted Key for read-only)
ApifyAPIFY_TOKENSame as your existing APIFY_TOKEN env var
