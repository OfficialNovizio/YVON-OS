2. Six Insight Layers — Stacked on Top of the Data

Each layer adds intelligence. Layer 1 is the data. Layer 6 is the recommendation engine.

🔢 Layer 1: Raw Metrics — the numbers
Health score, TS errors, API success rate, open issues, compression %. What the original plan already does.
📈 Layer 2: Trend Analysis — direction + velocity
Health score ↑3 this week (82→85→87). Projected: 92 by Friday if current velocity holds. Top contributor: API success improved from 97% → 99.8% after middleware fix.
TOON compression ↓0.04% since Monday. Drift rate: -0.01%/day. At this rate, falls below 99.9% threshold in 7 days. Triggered by PR #142 (47 new files).
DSA: Linear regression on snapshot ring buffer. Slope = velocity. R² = confidence. Simple math, no ML needed.
🔗 Layer 3: Root Cause Linking — what caused the dip?
TS errors spiked from 0→7 on Jun 18 14:32. Linked to commit d751303 by Dev (feat: VPS bridge). Files changed: lib/hermes-vps.ts, app/api/agent-ops/route.ts. Reverted in 6f0b3cb at 18:30. Recovery time: 4h.
API 500 spike on /api/scrape: 12 errors in 3 minutes at 08:00 UTC. Correlation: Apify rate limit (429 upstream). Agent: Kai (Instagram scrape). This endpoint accounts for 40% of all 500 errors this week.
DSA: Event correlation — join health dips with commit timestamps, agent activity, and deploy events. Hash-map lookup: commit SHA → agent → files changed.
🚨 Layer 4: Anomaly Detection — what's unusual?
Build duration anomaly: 3.2s → 47.8s at 02:00 UTC. Z-score: 4.8σ (threshold: 2.5σ). Caused by: V3 engine recompile with 512 merge iterations. Expected. Not an issue — flag dismissed.
API call volume spike: 1.2K → 4.7K requests between 14:00-15:00. Z-score: 3.1σ. Normal pattern: peak during working hours. Expected. Flag auto-dismissed by pattern learning.
DSA: Z-score on sliding window. μ and σ computed from last 7 days. |z| > 2.5 = anomaly. Pattern learning: if anomaly repeats at same time ±1h twice → auto-dismiss as "expected pattern."
📊 Layer 5: Comparative Context — how do you stack up?
TOON compression: 99.97% — top 5% across all tracked projects. Average: 99.5%. YVON-OS leads in token efficiency.
Issue resolution time: 2.4 days — below target of 1 day. 3 of 8 tracked projects resolve faster. Bottleneck: issues opened by Kai (marketing) take 4.2 days avg vs Dev (tech) at 0.8 days.
DSA: Cross-project percentile ranking. SELECT PERCENT_RANK() OVER (ORDER BY metric) FROM all repos. Updated hourly via snapshot roll.

4. Schema — What's New for v2 Insights

The 3 base tables stay. We add 2 intelligence tables + a recommendations view.

📋 Health Events Log

Every health-relevant event — commits, deploys, error spikes, fixes. Powers the timeline.

CREATE TABLE toongine_health_events ( id BIGSERIAL PRIMARY KEY, repo_id TEXT NOT NULL, event_type TEXT NOT NULL, -- commit|deploy|error_spike|fix|anomaly|pulse severity INT DEFAULT 0, -- 0=info 1=warn 2=critical title TEXT NOT NULL, detail TEXT DEFAULT '', linked_commit TEXT, -- commit SHA (nullable) linked_agent TEXT, -- agent who triggered it health_impact FLOAT DEFAULT 0, -- ± health points occurred_at TIMESTAMPTZ DEFAULT now() ); CREATE INDEX idx_events_repo_time ON toongine_health_events(repo_id, occurred_at DESC);
💡 Recommendations Cache

Pre-computed fix suggestions. Regenerated on every health score change.

CREATE TABLE toongine_recommendations ( id BIGSERIAL PRIMARY KEY, repo_id TEXT NOT NULL, priority INT NOT NULL, -- 0=P0 1=P1 2=P2 category TEXT NOT NULL, -- codebase|api|toon|issues title TEXT NOT NULL, detail TEXT DEFAULT '', impact_points FLOAT DEFAULT 0, -- estimated health gain effort_minutes INT DEFAULT 30, generated_at TIMESTAMPTZ DEFAULT now(), dismissed BOOLEAN DEFAULT false );
