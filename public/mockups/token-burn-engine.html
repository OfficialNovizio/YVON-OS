Each project gets a fixed number of pre-computed slots. Cron rolls them forward. Dashboard reads are instant.

6. DSA Summary — Why This Structure

Append-Only Log

Problem: Writes happen 100x more than reads. Updates would fragment.
Solution: Only INSERT. Never UPDATE. Never DELETE. Immutable audit trail.
Query: SELECT with composite index (project, created_at). O(log N).

Ring Buffer Snapshots

Problem: Dashboard aggregates 10K+ rows on every load. Slow.
Solution: Pre-compute fixed-size windows. Overwrite oldest slot.
Query: SELECT 24 rows. O(1) dashboard load. 66 rows per project total.

State Machine Provider

Problem: Provider changes break comparison. No history.
Solution: FSM with 5 valid transitions. Old providers stay as "Switched".
Query: SELECT WHERE is_current=true for active. WHERE state='switched' for history.

Min-Heap Leaderboard

Problem: Finding top-5 most expensive from 100K rows is expensive.
Solution: PostgreSQL ORDER BY + LIMIT uses B-tree internally (heap property).
Query: SELECT ... ORDER BY cost_usd DESC LIMIT 5. O(N log K) internally.
